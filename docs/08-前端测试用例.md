## 测试用例编号规则

格式：`TC-模块-序号`

- **TC**: Test Case（测试用例）
- **模块**: DASHBOARD, ROOMS, PLANTS, TASKS, COMMON
- **序号**: 001, 002, 003...

例如：`TC-ROOMS-001` 表示房间模块的第1个测试用例

## 优先级说明

| 优先级 | 说明 | 符号 |
|--------|------|------|
| P0 | 核心功能，必须通过 | 🔴 |
| P1 | 重要功能，应该通过 | 🟡 |
| P2 | 一般功能，可以延后 | 🟢 |
